# Tarot 模块回归检查清单

## 1) 基础路由与入口
- 打开首页 `/`，确认“塔罗牌”入口卡片可点击。
- 点击进入 `/divination/tarot`，页面可正常渲染（含顶部进度状态）。

## 2) 核心流程（必须）
1. **提问阶段**
   - 输入问题后“继续”按钮可用。
2. **牌阵阶段**
   - 可切换单张 / 三张 / 凯尔特十字。
3. **洗牌阶段**
   - 点击“洗牌并抽牌”后出现动画并进入揭牌。
4. **揭牌阶段**
   - 支持逐张翻牌。
   - “全部翻开”可一次揭示。
5. **解读阶段**
   - 点击“请求神谕”后开始流式输出。
   - 失败时展示错误态文案。

## 3) 配置兼容
- 默认配置模式：不启用自定义 API 时能使用后端默认配置。
- 自定义配置模式：`x-api-key` / `x-api-url` / `x-api-model` 能被请求带上。
- Tauri 模式：缺失配置时出现引导错误文案。

## 4) 历史记录
- 解读完成后写入 `tarot` 类型历史。
- 打开 `/history/tarot` 能看到新记录。
- 删除单条、清空全部功能可用。

## 5) UI 体验
- 牌面图片加载失败时有降级卡面。
- 逆位标签展示正常。
- 移动端顶部进度可展开/折叠。

## 6) 构建与静态检查
在 `frontend/` 下执行：

```bash
pnpm lint
pnpm build
```

要求：均成功，`build` 允许 chunk size 提示但不得失败。
